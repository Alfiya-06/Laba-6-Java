### 1. Класс Main

Основной класс проекта, демонстрирующий работу всех пользовательских аннотаций и их обработчиков.

Функции класса:

Создание тестовых объектов.

Вызов всех процессоров аннотаций.

Красивый вывод результатов в консоль.

Выполнение всех заданий по очереди.

Выполняемые задания:

Проверка аннотации `@Default`

Генерация строкового представления через `@ToString`

Вывод списка типов из аннотации` @Validate`

Чтение и вывод значений свойств аннотации `@Two`

### 2. Аннотация `@Default`

Аннотация для указания класса, используемого по умолчанию.

Характеристики:

Цель: TYPE, FIELD

Обязательное свойство: `Class<?> value()`

### 3. Класс DefaultProcessor

Обработчик аннотации `@Default`.

Основные методы:

`process(Object obj)` —
Определяет, присутствует ли аннотация, и выводит имя класса, указанного в `value()`.

Особенности:

Обрабатывает `null` с выводом сообщения об ошибке.

### 4. Аннотация `@ToString`

Аннотация, определяющая, нужно ли включать тип или поле в автоматически формируемое строковое представление объекта.

Характеристики:

Цель: TYPE, FIELD

Свойство:
`Mode value()` со значением по умолчанию: `Mode.YES`

Перечисление Mode:
`YES, NO`

Особенности:

Поля без аннотации включаются автоматически.

### 5. Класс ToStringGenerator

Генератор строкового представления объекта с учётом аннотации `@ToString`.

Отображает только поля:

аннотированные `@ToString(YES)`

или не аннотированные вообще

Исключает поля, помеченные0 `@ToString(NO)`

Возвращает строку "null" для null объекта

### 7. Класс ValidateProcessor

Обработчик аннотации `@Validate`.

Основной метод:

`process(Object obj)` —
выводит полные имена всех классов, указанных в свойстве `value()` аннотации.

Особенности:

Проверяет наличие аннотации

Обрабатывает `null` с сообщением об ошибке

### 8. Аннотация `@Two`

Аннотация с двумя обязательными свойствами.

Характеристики:

Цель: TYPE

Свойства:

`String first()`

`int second()`

### 9. Класс TwoProcessor

Обработчик пользовательской аннотации `@Two`.

Основной метод:

`process(Object obj)` —
читает аннотацию и выводит значения свойств `first` и `second`.

Особенности:

Проверяет наличие аннотации на классе

Обрабатывает null входные данные
